# Story 4: Create Backend Microservices Skeleton

## Goal

Create the basic backend service structure for foundation services.

## Tasks

- Create `services` directory.
- Create shared auth/middleware package if needed.
- Create `school-user-service`.
- Create `announcement-service`.
- Create `notification-service`.
- Create `file-service` placeholder.
- Add Dockerfile per service.
- Add `/health` endpoint per service.
- Add basic structured logging.
- Add README per service.

## Acceptance Criteria

- Each service can run locally.
- Each service exposes `/health`.
- Each service has its own Dockerfile.
- Service folders clearly show ownership and responsibility.

## Suggested Labels

`story`, `backend`, `microservices`, `foundation`
