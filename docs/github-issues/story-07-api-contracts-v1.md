# Story 7: Create API Contracts v1

## Goal

Agree endpoint shapes before frontend and backend teams build separately.

## Tasks

- Create `docs/api/api-contracts.md`.
- Define common API rules.
- Define auth header format.
- Define common success response format.
- Define common error response format.
- Define pagination format.
- Define tenant rule.
- Add foundation endpoint contracts.

## Foundation Modules

- Auth/profile
- Schools
- Users
- Classes
- Announcements
- Attendance
- Consent Forms
- Files
- Notifications
- Health check

## Acceptance Criteria

- Each endpoint includes method and URL.
- Each endpoint includes owning microservice.
- Each endpoint includes allowed roles.
- Each endpoint includes tenant rule.
- Each endpoint includes request body where applicable.
- Each endpoint includes success response.
- Each endpoint includes error responses.
- API contracts are reviewed by all team members.
