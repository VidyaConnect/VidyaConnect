Summary of Discussion

1. Panel Feedback  
 The university panel felt the proposed scope may be too small for a one-year industry project.  
 We agreed that the project should still keep a clear MVP, but also include extended features for the full-year delivery.  

2. Next Step: Feature Expansion  
 Students will think further about additional useful features that can be added beyond the MVP.  
 The goal is not to add random features, but to identify features that fit the product problem domain and can realistically be delivered.  

3. Student Tasks  
Students should:  
 Review the panel feedback.  
 Propose additional features for the one-year project scope.  
 Separate features into:  
   Core MVP  
   Extended scope  
   Optional/future scope  
 Prepare the SRS document based on their proposed scope.  
 Justify why each feature is included.  

4. Mentor Task: Ishan  
I will:

 Students and Mentor will compare both proposals.

 Update the final SRS based on the agreed scope.
